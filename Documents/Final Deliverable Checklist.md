# Final Deliverable Checklist

Please see other sections of the Capstone Handbook for further details and guidance. This list is provided for your convenience and to ensure that all items are completed and submitted. Please ensure all items are checked off before submission.

## Required final submission items (summary)
- **Your final capstone report** as a separate PDF document and **your lessons learned paper** as a separate PDF document (see Capstone Handbook: page 18 and Appendix 5).
- **Your final capstone presentation** as a PDF file.
- **Your client briefing** (includes an executive summary at the top) as a separate PDF document (see Capstone Handbook: page 19).
- **Your product** — fully functional prototype (FFP) or minimally viable product (MVP). Must be accessible via a public URL or provided as a VM/VM container or remotely accessible machine with login and access details for TA and instructors. The application must remain live and unchanged from submission until the course closes. Include under *Final Deliverables Access* in the report.

## The 13 final capstone checkpoint items
1. **Capstone report:** This is your final full capstone report.
2. **Feedback and review from your sponsor** on this checkpoint. Email correspondence with your sponsor, if applicable, is sufficient.
3. **List of functions and features implemented** in this checkpoint. This should be a section in the report.
4. **Source code extracted from the code repository and exported into a zip archive.** Provide as `FirstName_LastName_final_source_code_repository.zip`.
5. **Instructions for how to compile, build, and deploy your application.** Include as an appendix in the report.
6. **Code statistics** including but not limited to LOC, CLOC, cyclomatic complexity, cohesion, comment-to-code ratio, coupling, number of source modules, number of classes, number of methods, number of libraries, external components used, external libraries used, dependency tree, and programming languages used. Include as an appendix in the report.
7. **Link to code repository.** Ensure regular Git pushes; do not delete or alter commit history.
8. **Public link to your application or a VM image** that can be deployed and tested independently. Include under *Final Deliverables Access* in the report.
9. **Video showing the application being compiled, built, and deployed** (≤ 5 minutes, MP4). File name: `FirstName_LastName_compile_build_deploy.mp4`.
10. **Video explaining the design, architecture, and main modules** (≤ 5 minutes, MP4). File name: `FirstName_LastName_design_architecture_modules.mp4`.
11. **Video demonstrating application functionality** (≤ 5 minutes, MP4). File name: `FirstName_LastName_product_demo.mp4`.
12. **Known problems, gaps, defects, and mitigation / future work plan.** Include as a section in your report.
13. **Differences between this checkpoint and the previous one,** including changes in items 3, 6, and 12. Include in an appendix.
